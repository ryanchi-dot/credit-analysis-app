# 对话功能使用指南

## 🎉 新功能：对话模式

现在小程序支持**对话交互**了！你可以：
- ✅ 连续提问
- ✅ 追问细节
- ✅ 对话式交互
- ✅ 自动保存对话历史

---

## 🚀 如何使用

### 进入对话模式

有两种方式进入对话模式：

#### 方式1：从首页进入
1. 打开小程序首页
2. 滚动到底部，看到"想要深入了解更多？"
3. 点击"进入对话模式"

#### 方式2：直接访问
1. 在小程序导航中点击"对话"
2. 直接进入对话页面

## 📱 功能特点

### 1. 智能记忆
- AI 会记住之前的对话内容
- 可以基于上下文进行回复
- 支持多轮连续对话

### 2. 自动保存
- 对话历史自动保存
- 关闭小程序后重新打开，对话历史不会丢失
- 支持跨设备同步（如果部署到云）

### 3. 清空历史
- 点击"清空对话"按钮
- 清空所有对话记录
- 重新开始新的对话

---

## 🔄 对话 vs 单次分析

### 对话模式
```
适合：深入了解、多轮提问
优点：
- 可以连续提问
- 记忆对话上下文
- 交互灵活
示例：
  用户：分析腾讯
  AI：[生成报告]
  用户：利润多少？
  AI：1941亿元
  用户：风险呢？
  AI：[详细风险分析]
```

### 单次分析模式
```
适合：快速获取完整报告
优点：
- 一次性生成完整报告
- 内容详尽
- 适合归档
示例：
  用户：分析腾讯
  AI：[生成完整报告]
```

## ⚙️ 技术实现

### 后端
- 新增 API 接口：
  - POST /api/chat - 发送消息
  - GET /api/chat/{session_id} - 获取历史
  - DELETE /api/chat/{session_id} - 清空历史
- 支持多会话管理
- 使用 Agent 的记忆功能

### 前端
- 聊天界面（类似微信聊天）
- 实时显示对话
- 自动滚动到最新消息
- 支持清空对话历史

---

## 📊 对话流程

```
1. 用户输入消息
   ↓
2. 小程序调用 POST /api/chat
   ↓
3. 后端处理（使用 session_id 保持记忆）
   ↓
4. Agent 生成回复
   ↓
5. 返回给小程序
   ↓
6. 显示在聊天界面
```

## ⚠️ 注意事项

1. **会话有效期**
   - 对话历史保存在后端服务器
   - 如果后端重启，历史可能会丢失（生产环境建议使用数据库）

2. **记忆限制**
   - Agent 记忆窗口：最近 20 轮对话
   - 超过 20 轮的旧对话会被自动清除

3. **响应时间**
   - 首次提问可能需要 2-5 分钟
   - 后续提问通常更快（1-3 分钟）

## ❓ 常见问题

### Q1: 对话历史会保留多久？
A: 取决于后端配置，当前版本保存在内存中，服务器重启后会丢失。建议定期清空或导出重要对话。

### Q2: 可以同时分析多家公司吗？
A: 当前版本一个会话只能分析一家公司。如需分析多家公司，请使用"清空对话"功能，或等待后续多主题对话功能上线。

### Q3: 对话模式和单次分析模式有什么区别？
A: 对话模式支持连续提问，单次分析模式一次性生成完整报告。根据你的需求选择。

### Q4: 如何开始新的对话？
A: 点击"清空对话"按钮，或者等待 20 轮对话后自动清除旧记忆。
